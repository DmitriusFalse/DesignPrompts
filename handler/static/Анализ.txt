АНАЛИЗ ДИЗАЙНА: Модальное окно «Холсты»
Danbooru Prompt Builder — оценка нового функционала
═══════════════════════════════════════════════════════════════════════════

ЧАСТЬ 1: ЧТО РЕАЛИЗОВАНО
──────────────────────────

✓ МОДАЛЬНОЕ ОКНО ДЛЯ УПРАВЛЕНИЯ ХОЛСТАМИ
  Холст = запись промпта + сгенерированное изображение.
  Это правильная концепция: пользователь может сохранить результат
  генерации и вернуться к нему позже.

✓ СПИСОК ХОЛСТОВ СЛЕВА
  Показывает название холста («test») и дату создания.
  Есть кнопка удаления (иконка корзины).

✓ КНОПКИ ДЕЙСТВИЙ
  • «Копировать» — вероятно, копирует промпт в буфер обмена
  • «Открыть» — загружает холст в основной интерфейс
  • «Закрыть» — закрывает модальное окно

✓ ЗАГЛУШКА «NO IMAGE GENERATED»
  Если изображение не сгенерировано, показывается иконка камеры
  с текстом. Это лучше, чем пустое место.


ЧАСТЬ 2: КРИТИЧЕСКИЕ ПРОБЛЕМЫ
──────────────────────────────

ПРОБЛЕМА 1: ОГРОМНОЕ КОЛИЧЕСТВО ПУСТОГО ПРОСТРАНСТВА
──────────────────────────────────────────────────────
Модальное окно занимает ВЕСЬ экран (или почти весь), но контент
занимает ~10% площади. Остальные 90% — пустой тёмный фон.

Это выглядит как незавершённая разработка. Пользователь видит
огромное окно и думает «что-то сломалось» или «где остальной контент?».

Рекомендация:
• Уменьшить размер модального окна до 800×600px (или 70% экрана)
• Центрировать окно по вертикали и горизонтали
• Добавить затемнение фона (overlay) за окном

ПРОБЛЕМА 2: РАСПОЛОЖЕНИЕ ЭЛЕМЕНТОВ ХАОТИЧНО
────────────────────────────────────────────
• Изображение-заглушка: левый верхний угол, ~200×200px
• Поле ввода: под изображением, растянуто на всю ширину
• Кнопка «Копировать»: по центру, под полем ввода
• Кнопка «Открыть»: правый край, на одном уровне с «Копировать»
• Кнопка «Закрыть»: левый нижний угол

Это не следует ни одному паттерну размещения. Кнопки разбросаны
по разным углам, пользователь не знает куда смотреть.

Рекомендация:
Стандартная компоновка модального окна:
┌─────────────────────────────────────────────────────────────────────┐
│  Холсты                                              [—] [□] [×]   │
├──────────────┬──────────────────────────────────────────────────────┤
│              │                                                      │
│  Список      │         [ИЗОБРАЖЕНИЕ ХОЛСТА]                         │
│  холстов     │         400×400px, по центру                         │
│  (200px)     │                                                      │
│              │   Название: [test________________]                   │
│  • test      │   Дата: 2026-06-21 16:44:55                          │
│    21.06     │                                                      │
│  • ...       │   [Копировать промпт]  [Открыть холст]  [Удалить]   │
│              │                                                      │
├──────────────┴──────────────────────────────────────────────────────┤
│  [Закрыть]                                                          │
└─────────────────────────────────────────────────────────────────────┘

ПРОБЛЕМА 3: СПИСОК ХОЛСТОВ СЛИШКОМ УЗКИЙ
─────────────────────────────────────────
Левая панель ~100px шириной. Название «test» помещается, но длинные
названия обрежутся. Дата показана полностью (ISO формат), но это
занимает много места.

Рекомендация:
• Ширина панели: 200-250px
• Формат даты: «21.06.2026 16:44» вместо ISO
• Название холста: 1 строка, обрезка с многоточием
• Hover: подсветка строки
• Активный холст: акцентный цвет фона

ПРОБЛЕМА 4: НЕТ ИНФОРМАЦИИ О ПАРАМЕТРАХ ГЕНЕРАЦИИ
───────────────────────────────────────────────────
Холст должен хранить не только промпт и изображение, но и параметры:
• Модель
• Шаги (STEPS)
• CFG
• Сид
• Разрешение
• Семплер
• Шедулер

Без этой информации пользователь не сможет воспроизвести результат.

Рекомендация:
Добавить секцию «Параметры генерации» под изображением:
─────────────────────────────────────────────────────────────────────
│  Параметры:                                                         │
│  Модель: PONY\prefectPonyXL_v6-20fe.safetensors                    │
│  Разрешение: Portrait Tall 4:7 – 768x1344                          │
│  Шаги: 25 | CFG: 8 | Сид: 737168481                                │
│  Семплер: dpmpp_2m_sde_gpu | Шедулер: karras                       │
└─────────────────────────────────────────────────────────────────────┘

ПРОБЛЕМА 5: КНОПКА «ЗАКРЫТЬ» В НЕПРАВИЛЬНОМ МЕСТЕ
───────────────────────────────────────────────────
Кнопка «Закрыть» расположена в левом нижнем углу. Это нестандартно.

Стандартные паттерны:
• Кнопка [×] в правом верхнем углу шапки (уже есть ✓)
• Кнопка «Закрыть» в правом нижнем углу (рядом с другими действиями)
• Закрытие по клавише Escape

Рекомендация:
• Убрать кнопку «Закрыть» из левого нижнего угла
• Оставить только [×] в шапке
• Добавить закрытие по Escape
• Если нужна кнопка — разместить в правом нижнем углу

ПРОБЛЕМА 6: ПОЛЕ ВВОДА БЕЗ ПОДПИСИ
────────────────────────────────────
Поле с текстом «test» не имеет label. Пользователь не понимает,
что это: название холста? промпт? комментарий?

Рекомендация:
Добавить label «Название холста:» или «Промпт:» над полем.
Если это промпт — сделать textarea для многострочного текста.

ЧАСТЬ 4: ФУНКЦИОНАЛЬНЫЕ УЛУЧШЕНИЯ
──────────────────────────────────

1. ПОИСК ПО ХОЛСТАМ
   Поле поиска в левой панели для фильтрации по названию или дате.

2. СОРТИРОВКА
   Сортировка по дате (новые/старые), по названию, по размеру файла.

3. ПРЕВЬЮ В СПИСКЕ
   Показать миниатюру изображения рядом с названием холста в списке.
   Это поможет быстрее найти нужный холст.

4. МАССОВЫЕ ДЕЙСТВИЯ
   Чекбоксы для выбора нескольких холстов → массовое удаление/экспорт.

5. ЭКСПОРТ/ИМПОРТ ХОЛСТОВ
   Кнопка «Экспортировать» для сохранения холста в файл (JSON + изображение).
   Кнопка «Импортировать» для загрузки холста из файла.

6. РЕДАКТИРОВАНИЕ НАЗВАНИЯ
   Двойной клик по названию холста → редактирование inline.

7. ТЕГИ ДЛЯ ХОЛСТОВ
   Возможность добавлять теги к холстам для организации (например,
   «персонаж_А», «пейзаж», «эксперимент»).


ЧАСТЬ 5: ОЦЕНКА
───────────────

Концепция холстов: ★★★★★ (5/5)
  Отличная идея — сохранять промпт + изображение + параметры.

Текущая реализация: ★★☆☆☆ (2/5)
  Очень сырая версия. Много пустого пространства, хаотичное
  расположение элементов, отсутствует важная информация.

UI/UX: ★★☆☆☆ (2/5)
  Не следует стандартным паттернам модальных окон. Кнопки разбросаны,
  нет визуальной иерархии.

Общая оценка: 2.3/5

ИТОГ
═══════════════════════════════════════════════════════════════════════════

Концепция холстов правильная и полезная. Но текущая реализация
выглядит как ранний прототип — много пустого пространства, хаотичное
расположение элементов, отсутствует важная информация (параметры
генерации).

Главная задача: переделать компоновку модального окна по стандартному
паттерну (список слева, детали справа, действия внизу). Уменьшить
размер окна, добавить информацию о параметрах, улучшить UX списка.

После этих изменений функционал холстов станет полноценным инструментом
для управления результатами генерации.
